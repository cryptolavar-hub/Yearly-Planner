This folder is the persistent knowledge base for this repository. It is intended to prevent repeated mistakes, preserve decisions, and make future fixes faster and safer.

## How this KB is used

- **Truth source**: Prefer code + logs + this KB over older free-form notes.
- **Session-based updates**: Each working session should produce or update:
  - a bug/issue register entry (if new issues discovered)
  - a short session log (what changed, what was learned, what is next)
- **Approval workflow**: No code changes are executed until CryptoLavar approves the proposed fix plan and impact analysis.

## Documents

- `KB_ROLES_SKILLSETS.md`
  - Required roles, skillsets, and the persona roster used to execute work under direction.
- `KB_BUG_AUDIT_AND_EXECUTION_PLAN.md`
  - Bug register, recommended fix options with pros/cons/impact, and a staged execution plan (approval-gated).
- `SESSION_LOG_2025-12-18.md`
  - What was reviewed, key findings, and the proposed next actions for approval.
- `../ENVIRONMENT_VARIABLES.md`
  - Central list of runtime configuration knobs for backend/frontend (no `.env` committed).
